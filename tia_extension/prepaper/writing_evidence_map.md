## Claims To Remove Or Hold

| Potential claim | Reason to remove or hold |
|---|---|
| The workflow proves hardware stability. | No hardware measurement exists. Vendor SPICE macromodels are simulation evidence only. |
| The noise results are measured or detector validated. | Current noise outputs are first-pass behavioural estimates using configured assumptions. |
| The Safe/Marginal/Risky labels are universal TIA classifications. | Labels are project-defined screening categories. |
| The method is a new TIA topology. | Literature includes many TIA topology contributions; this project contributes a reproducible screening workflow. |
| Detailed Cherry-Hooper performance comparisons. | The strongest Cherry-Hooper candidates are currently metadata-only or DOI-only unless full text is obtained later. |
| Quantitative claims from abstract-only DOI candidates. | Abstract-only and metadata-only candidates are not full-text reading evidence. |

## Immediate Writing Use

- Introduction can use full-text sources to motivate the coupled effects of photodiode capacitance, compensation, bandwidth, peaking, and noise.
- Related Work can organize topology families using full-text papers and flag metadata-only Cherry-Hooper/high-speed sources as "to verify."
- Methodology and Results should rely primarily on project evidence and avoid importing external performance numbers unless the source has full text and the number is checked.
- Limitations should explicitly state no hardware validation, no measured-noise validation, and no final Q3 submission-readiness claim.
